# Web Application Template

Opinionated React + TypeScript template using:

- Vite 7
- TanStack Router (file-based routing + auto-generated route tree)
- TanStack Query
- Tailwind CSS v4
- shadcn/ui (Radix Nova style)
- ESLint 9 with type-aware TypeScript linting
- Prettier + lint-staged + Husky pre-commit hooks
- Supabase local development config

## Requirements

- Node.js 20+ (current LTS recommended)
- npm 10+
- Optional: Supabase CLI (for local database/auth/storage stack)

## Quick Start

```bash
npm install
npm run dev
```

App runs on Vite's default dev URL (usually `http://localhost:5173`).

For the Supabase example, copy [`.env.example`](./.env.example) to `.env` and fill in
your project values.

## Scripts

- `npm run dev`: start Vite dev server
- `npm run build`: type-check and produce production build
- `npm run preview`: serve the production build locally
- `npm run lint`: run ESLint across the repo
- `npm run prepare`: install Husky hooks

## Routing

This template uses TanStack Router with the Vite plugin:

- Route files live in `src/routes`.
- `src/routeTree.gen.ts` is generated automatically from route files.
- `routeTree.gen.ts` is intentionally excluded from lint/format.

Create a new route by adding a file in `src/routes`, for example:

```tsx
import { createFileRoute } from "@tanstack/react-router";

export const Route = createFileRoute("/settings")({
  component: () => <div>Settings page</div>,
});
```

## Styling and UI

- Tailwind CSS v4 is configured in `src/index.css`.
- `@fontsource-variable/geist` is loaded globally.
- shadcn/ui is configured via `components.json`.
- Alias `@` resolves to `src` in both Vite and TypeScript.

## Data Fetching

- `QueryClient` is provided at the root route.
- React Query devtools are lazy-loaded and only included during development builds.

## Minimal Library Examples

The home route at [`src/routes/index.tsx`](./src/routes/index.tsx) intentionally keeps
examples as small as possible while still exercising the installed runtime libraries:

- `@tanstack/react-router`: file-based routes for `/` and `/about`
- `@tanstack/react-query`: one `useQuery` example
- `@tanstack/react-form`: one controlled field and submit handler
- `@tanstack/zod-form-adapter` + `zod`: one schema-backed validator
- `@supabase/supabase-js`: one browser client in [`src/lib/supabase.ts`](./src/lib/supabase.ts)
- `shadcn` + `radix-ui`: the shared Button component in `src/components/ui/button.tsx`
- `class-variance-authority`: button variants
- `clsx` + `tailwind-merge`: the `cn()` helper in `src/lib/utils.ts`
- `lucide-react`: icons on the example cards/buttons
- `tw-animate-css`: the animated hero section
- `@fontsource-variable/geist`: global font import in `src/index.css`

The remaining packages are build/config tooling rather than runtime libraries:

- Vite + `@vitejs/plugin-react` + `@tailwindcss/vite`
- TypeScript + `@types/*`
- ESLint stack (`@eslint/js`, `typescript-eslint`, `eslint-plugin-react-hooks`,
  `eslint-plugin-react-refresh`, `eslint-config-prettier`, `globals`)
- Prettier + `lint-staged` + Husky
- Supabase CLI
- TanStack Router Vite plugin and devtools packages

## Linting and Formatting

- ESLint is configured with:
  - JS recommended rules
  - TypeScript type-aware rules (`recommendedTypeChecked`)
  - React Hooks rules
  - React Refresh rules
- Prettier is applied through `lint-staged` on staged files.
- Husky pre-commit hook runs `npx lint-staged`.

## Supabase Local Development

`supabase/config.toml` is included for local stack setup (API, DB, auth, storage, studio).

Browser env vars are read from `.env` by Vite:

```env
VITE_SUPABASE_URL=...
VITE_SUPABASE_ANON_KEY=...
```

Typical commands:

```bash
npx supabase start
npx supabase status
npx supabase stop
```

If you use seed-on-reset (`[db.seed]` in `config.toml`), create `supabase/seed.sql` or update the configured seed path.

## Template Conventions

- Keep generated files generated (`src/routeTree.gen.ts`).
- Prefer importing app modules with `@/` aliases.
- Keep `QueryClient` singleton outside render functions.
- Gate dev-only tools and diagnostics from production bundles.
